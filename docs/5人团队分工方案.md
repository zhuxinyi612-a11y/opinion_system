# 5人团队分工方案

贴合舆情分析系统技术栈，基础功能全覆盖、高级功能均衡分摊

## 人员整体划分

爬虫开发 1 人、数据处理 \& NLP 2 人、时序算法预测 1 人、可视化 \& 系统整合 1 人

## 分工明细

### 1 号：爬虫工程师（数据采集全链路）

对应后端模块：网络爬虫

1. 主流新闻网站、社交平台爬虫开发（静态网页 \+ 异步接口、反爬处理）

2. 多源数据统一入库（MySQL/Elasticsearch）

3. 定时爬取、增量采集、数据源拓展维护

4. 配合 2 号输出原始采集数据

### 2 号：数据预处理工程师（数据清洗 \+ 文本基础处理）

对应后端模块：数据清洗及预处理、内容分析

1. 原始数据去重、过滤噪点、统一字段格式标准化

2. 网页正文抽取算法开发，剔除广告、导航栏等冗余内容

3. 中文分词、停用词过滤、文本特征提取（TF\-IDF / 词向量）

4. 输出标准化清洗文本数据集，供给 3、4 号使用

### 3 号：NLP 语义分析工程师（舆情分类、聚合、情感分析）

对应后端模块：热点事件发现、舆情事件聚合、情感倾向分析；

1. 训练机器学习舆情主题分类模型，自动识别热点事件

2. 实现文本相似度算法，完成报道聚合、相似事件检索

3. 搭建情感计算模型，区分文本正面 / 负面 / 中立情感立场


### 4 号：时序算法工程师（舆情生命周期预测 \+ 传播溯源）

对应后端模块：事件舆情预测；高级功能：事件溯源与传播路径图

1. 基于发布时间、报道量构建时序模型，划分舆情生命周期：潜伏期 / 成长期 / 高潮期 / 衰退期

2. 识别传播关键账号、大 V、官方媒体介入节点

3. 设计事件传播链路图数据结构，输出传播节点数据

4. 迭代优化舆情发展趋势预测模型
5. 开发高级功能：虚假文本判别模型，输出内容真实置信度

### 5 号：前后端整合 \& 可视化开发（系统整体搭建 \+ 创新设计）

1. 搭建 Web 后端服务接口，打通爬虫、清洗、NLP、算法各模块数据流转

2. 开发可视化页面：热点榜单、情感走势图、舆情生命周期曲线、传播路径图谱

3. 开发历史事件检索、数据查询交互功能

4. 项目整体联调、bug 修复、项目文档编写、拓展系统创新功能

## 分工优势

1. 刚好匹配 5 人人数限制，技术职责边界清晰，无重复工作

2. 7 项基础后端功能全部覆盖，2 项高级功能分摊至 3、4 号成员，工作量均衡

3. 技术方向独立细分：爬虫工程、数据预处理、NLP 语义、时序算法、全栈可视化，每人专注单一模块，支持并行开发

4. 5 号统筹整套系统，负责整体联调与创新拓展，便于项目收尾整合
